### Подход 1: NeMo + PyAnnote

#### **Плюсы:**

1.  **Высокое качество транскрибации от NeMo:**
    *   NeMo предлагает множество предобученных моделей (QuartzNet, CitriNet, Conformer), которые часто показывают лучшее качество на сложных данных (с шумом, технической лексикой) по сравнению со стандартной Whisper.
    *   Возможность **тонкой настройки** на своих данных. Это ключевое преимущество, если у вас есть размеченный датасет под вашу предметную область.
    *   Поддержка **языковой модели** для исправления ошибок и улучшения качества текста, особенно для специфической терминологии.

2.  **Гибкость и контроль:**
    *   Можно независимо выбирать и улучшать каждую часть конвейера. Например, можно заменить модель диаризации или ASR-модель на более подходящую для вашей задачи.
    *   Полный контроль над порогом паузы для сегментации, что позволяет тонко настраивать процесс под длину реплик в ваших данных.

#### **Минусы:**

1.  **Сложность настройки и эксплуатации:**
    *   Необходимо объединять две независимые системы, писать код для их взаимодействия и управления данными между ними. Это создает более сложный пайплайн для развертывания и поддержки.

2.  **Накопление ошибок (Error Propagation):**
    *   Ошибка на этапе диаризации (например, пропуск короткой реплики или неверное определение говорящего) не может быть исправлена на этапе транскрибации и финально войдет в результат.

### Подход 2: Whisper + PyAnnote

#### **Плюсы:**

1.  **Простота использования и скорость:**
    *   Whisper невероятно прост в использовании. Его API элементарен для базовой транскрибации.
    *   Одна модель обрабатывает множество языков и задач, не требуя указания языка.
    *   Обычно работает быстрее, чем тонко настроенные модели NeMo, особенно в "out of the box" сценарии.

2.  **Отличное качество "из коробки":**
    *   Whisper показывает блестящие результаты на разнообразных данных без дообучения. Он очень устойчив к фоновому шуму, акцентам и хорошо обобщает.

#### **Минусы:**

1.  **Ограниченные возможности для дообучения:**
    *   Хотя технически Whisper можно дообучить, этот процесс сложнее и менее документирован, чем для NeMo. Сообщество NeMo имеет больше опыта и инструментов для тонкой настройки.

2.  **Отсутствие глубокой интеграции с языковыми моделями:**
    *   Whisper в своей основе — трансформер, который уже включает подобие языковой модели, но у него нет такой же простой возможности подключить внешнюю N-gram или трансформерную LM, как в NeMo.


#### **Результаты**
    1. Подходы показали одинаковые результаты транскрибации и диаризации. Это связано с качеством аудиозаписи, где дикторы отлично произносят слова, на заднем фоне отсутствуют лишние шумы.<br>
    2. WER = 0, CER = 0 - метрики для транскрибации<br>
    3. DER = 0.07 (7%) для Pyannote, что также является отличным результатом
